# Issue/PR運用・テンプレートガイド

このドキュメントは、pantry-plannerプロジェクトのIssue/PR運用ルール・テンプレート・記載例をまとめたものです。

## 1. Issue運用

- バグ・機能追加・改善提案は必ずIssue化
- タイトル・内容・再現手順・期待動作を明記
- ラベル・担当者を必ず設定

### Issueテンプレート例

```
## 概要

## 再現手順

## 期待動作

## 補足
```

## 2. PR運用

- 1PR 1目的・小さな粒度を推奨
- 目的・変更内容・影響範囲・テスト結果を明記
- 関連Issueを必ずリンク

### PRテンプレート例

```
## 目的

## 変更内容

## 影響範囲

## テスト結果

## 関連Issue
```
